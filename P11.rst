Факторы, которые на данном этапе не позволяют реализовать индикатор
===================================================================

На данный момент нет данных о приложенных к заявкам файлах в формате открытых данных OCDS. Если такие данные станут доступны, сможем реализовать данный индикатор.


***************
Суть индикатора
***************

Аналитическая система проверяет наличие файлов , приложенных к выигравшей конкурсной заявке.


****
Риск
****
Риск сговора между представителями закупающей организации и участником конкурса. Использование "своего поставщика" в коррупционных закупках.

*******************************
Нарушение норм/принципов закона
*******************************

Несоблюдение закупающими организациями при оценке конкурсных предложений положений статьи 29 Закона КР «О государственных закупок», когда присуждение договора осуществляется участнику конкурса, не соответствующему минимальным квалификационным требованиям или предложенная им продукция, не соответствовала требованиям технической спецификации (отсутствие письма- подтверждения завода-изготовителя, отсутствие необходимых сертификатов и т.д.)

*******************************
Методология расчета индикатора.
*******************************

Уровень расчета
===============
Индикатор расcчитывается на уровне *процедуры*.

Источники данных для расчета
============================

Для расчета индикатора используются следующие источники данных:

- API системы государственных закупок в OCDS формате

Типы процедур
=============

Индикатор рассчитывается для следующих типов процедур:

- метод прямого заключения договора,
- одноэтапный,
- двухэтапный,
- упрощенный,
- на понижение цены.


Статусы процедур
----------------

Индикатор расчитывается для процедур, которые:

- находятся в статусе ``complete``


Частота расчета
===============

Если выполнены все условия для активации расчета индикатор, он рассчитывается один раз.
